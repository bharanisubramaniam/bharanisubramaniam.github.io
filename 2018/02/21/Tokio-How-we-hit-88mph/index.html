Tokio:How we hit 88mph

the tech talk
2018-02-21

RustFest Zürich 2017 - Tokio: How we hit 88mph by Alex Crichton

Background

It is costly to have synchronous IO operations in server applications. Rust used to have green threads but was removed by this RFC becuase it was difficult to implement green threads with out a runtime or FFI cost. So eventually the only way to write scalable server application is by using asynch IO.

Why not just use Metal IO?

Metal IO or mio is a library that provides asynch IO capabilities on top of many operating systems. But mio is a low level framework and writing higher level applications with just mio is tricky.

Don’t worry - Future is bright

This problem is solved in many languages with futures and asynch/await syntax. In Rust the futures crate provides zero cost abstractions to conviniently write asynch IO

What is a future?

It is a sentinel for a value that is being computed. So in essence a future is a value that is not ready yet.

What can i do with future?

From Aaron Turon’s blog on futures

Sequential composition:

f.and_then(|val| some_new_future(val)). Gives you a future that executes the future f, takes the val it produces to build another future some_new_future(val), and then executes that future.

Mapping:

f.map(|val| some_new_value(val)). Gives you a future that executes the future f and yields the result of some_new_value(val).

Joining:

f.join(g). Gives you a future that executes the futures f and g in parallel, and completes when both of them are complete, returning both of their values.

Selecting:

f.select(g). Gives you a future that executes the futures f and g in parallel, and completes when one of them is complete, returning its value and the other future. (Want to add a timeout to any future? Just do a select of that future and a timeout future!)

So what is tokio?

Tokio is a fusion of MIO and Futures with an event loop.Tokio makes it convinient to write fast networking code in Rust

What can you do with Tokio now?

Traits for
Single valued - Future
Stream of value - Stream
Pushing value - Sink
One shot channels
compute inmemory value asynchronously
mpsc - Multiple Producer Single Consumer
Integration with Rayon
